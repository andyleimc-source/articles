# 第三阶段：核心概念

## 3.1 CLI 基础和常用命令

### 启动与退出

```bash
# 启动 Claude Code（当前目录作为 workspace）
claude

# 指定工作目录
claude /path/to/project

# 带系统提示启动
claude --system "You are a Python expert"

# 单行模式：不进入交互界面，直接执行一条指令后退出
claude "解释这段代码"

# 退出
/exit
# 或 Ctrl+C
```

### CLI 启动参数（claude --help）

| 参数 | 说明 |
|------|------|
| `claude -h` | 查看完整帮助 |
| `claude --version` | 查看版本 |
| `claude -p` / `claude --print` | 非交互模式：输出结果后直接退出，适合管道/脚本 |
| `claude --system-prompt PROMPT` | 启动时注入系统级提示 |
| `claude /path` | 指定 workspace 目录 |

### 核心命令一览（/xxx 系列）

Claude Code 交互界面内输入 `/` 触发命令：

| 命令 | 作用 |
|------|------|
| `/help` | 查看所有命令帮助 |
| `/models` 或 `/model` | 切换模型（思考模型/快速模型/混合） |
| `/reasoning` | 开启/关闭思考模式（Deep Research 推荐开启） |
| `/clear` | 清屏（清除当前会话显示，不影响上下文） |
| `/memory` | 查看当前会话加载的记忆文件列表 |
| `/exit` 或 `/quit` | 退出 CLI |
| `/compact` | 压缩上下文，释放 Token 空间 |
| `/commit` | 提交 Git 改动（需安装 superpowers 插件） |
| `/resume` | 恢复历史会话 |

### 命令行参数传递

```bash
# 单次对话（不进入交互模式）
claude "解释这段代码"

# 传入文件内容
claude "review this file: $(cat script.py)"

# 使用 heredoc 传入多行
claude << 'EOF'
Write a Python function that:
1. Takes a list of numbers
2. Returns the sum
3. Handles empty list
EOF
```

### exec 命令详解

`exec` 是 Claude Code 最强大的命令之一，用于执行系统命令：

```bash
# 基本语法
exec(command="ls -la")

# 指定工作目录
exec(command="git status", workdir="/path/to/project")

# 后台运行
exec(command="npm run dev", background=true)

# 超时设置（秒）
exec(command="python train.py", timeout=300)
```

### 输出解析方法

```bash
# 管道传递
exec(command="ls -la | grep '.py'")

# 重定向到文件
exec(command="echo 'log output' > /tmp/log.txt")

# 获取命令返回值
exec(command="git rev-parse HEAD")
```

## 3.2 目录管理与会话

### workspace 工作目录

Claude Code 的工作目录是所有文件操作的根目录：

```
默认：当前启动目录
可通过启动参数指定：claude /path/to/project
```

### 绝对路径 vs 相对路径

```bash
# 绝对路径（从根目录开始）
/Users/andy/Documents/project/main.py

# 相对路径（从当前 workspace 算起）
./src/utils.py
../config/settings.json
~/Documents/project/main.py  # ~ 代表用户主目录
```

### Main Session vs Sub Session

| 类型 | 说明 |
|------|------|
| **Main Session** | 主会话，长期记忆和上下文 |
| **Sub Session** | 子会话，隔离的临时任务 |

```bash
# Main Session：直接与 Claude 对话的主会话
# 拥有完整上下文：SOUL.md、AGENTS.md、MEMORY.md

# Sub Session：通过 sessions_spawn 启动的隔离会话
# 用途：并行执行多个任务，互不干扰
```

### 会话持久化与上下文继承

Claude Code 通过 `Agent` 工具启动并行子 Agent：

```
# 在对话中让 Claude 并行执行多个任务
"同时分析 config.json 和 README.md 这两个文件"
# Claude 会自动启动多个 Agent 并行处理，结果汇总回主会话
```

> ⚠️ `sessions_spawn` 是 OpenClaw 的语法，Claude Code 不支持此命令。

## 3.3 上下文管理系统

Claude Code 通过两类文件管理上下文：

```
┌─────────────────────────────────┐
│         System Prompt            │  ← Claude 系统级指令（固定）
├─────────────────────────────────┤
│         CLAUDE.md               │  ← 项目/用户自定义指令
├─────────────────────────────────┤
│    memory/ 自动记忆文件           │  ← 跨会话持久记忆（插件管理）
├─────────────────────────────────┤
│        当前会话消息               │  ← 本次对话内容
└─────────────────────────────────┘
```

> ⚠️ `SOUL.md`、`AGENTS.md`、`USER.md`、`HEARTBEAT.md` 是 **OpenClaw** 的体系，不是 Claude Code 原生功能。

### CLAUDE.md — 项目指令文件

Claude Code 启动时自动读取工作目录下的 `CLAUDE.md`，也会读取全局的 `~/.claude/CLAUDE.md`。

```markdown
# CLAUDE.md 示例

## 项目说明
这是一个 FastAPI 后端项目。

## 编码规范
- Python 用 Black 格式化
- 所有函数必须有类型注解
- Git 用 rebase 而非 merge

## 常用命令
- 启动服务：uvicorn main:app --reload
- 运行测试：pytest
```

**加载优先级（从高到低）：**

| 位置 | 文件 | 作用 |
|------|------|------|
| 工作目录 | `CLAUDE.md` | 当前项目指令 |
| 父目录 | `CLAUDE.md` | 父级项目指令 |
| 全局 | `~/.claude/CLAUDE.md` | 跨项目通用指令 |

### memory/ — 跨会话记忆

Claude Code 配合记忆插件（如 superpowers auto-memory）可以在会话间保留记忆：

```
~/.claude/projects/<project>/memory/
├── MEMORY.md        # 记忆索引
├── user_profile.md  # 用户偏好
├── feedback_xxx.md  # 行为反馈
└── project_xxx.md   # 项目上下文
```

记忆由 Claude 自动写入，跨会话自动加载，无需手动维护。

## 3.4 Skills 技能系统

### 什么是 Agent Skills

Skills 是预定义的工具和流程封装，让 Claude 能调用特定功能：

```
Skills = 工具定义 + 使用说明 + 执行逻辑
```

### 常用 Skills 一览（需安装插件）

Skills 不是内置的，均来自用户安装的插件。以下是常见可选 Skills：

| Skill | 用途 |
|--------|------|
| github | GitHub 操作（issues, PR, CI）|
| imsg | iMessage 发送 |
| apple-reminders | 苹果备忘录 |
| 1password | 密码管理 |
| superpowers | 开发工作流（commit、review、TDD 等）|
| baoyu-skills | 内容创作、图片生成、社媒发布 |

### 如何使用一个 Skill

```bash
# 查看可用的 skills
ls ~/.agents/skills/

# 查看某个 skill 的说明
cat ~/.agents/skills/github/SKILL.md
```

### 自定义 Skill 结构

```
~/.agents/skills/my-skill/
├── SKILL.md          # 核心定义文件
├── scripts/          # 辅助脚本
└── references/       # 参考文档
```

### SKILL.md 基础结构

```markdown
---
name: my-skill
description: >
  这个技能做什么
  使用场景：xxx
---

# My Skill

## 使用方法

1. 步骤一
2. 步骤二

## 示例

/使用 my-skill 完成 xxx
```

## 3.5 MCP（Model Context Protocol）

### 什么是 MCP

MCP 是一种**协议标准**，让 AI 模型能连接外部工具和数据源：

```
┌─────────────┐       MCP        ┌──────────────┐
│  Claude AI  │ ←──────────────→ │  MCP Server  │
│   Model     │                  │              │
└─────────────┘                  └──────────────┘
                                         ↓
                              ┌──────────────┐
                              │   数据源      │
                              │ 文件/API/DB  │
                              └──────────────┘
```

### 已安装 MCP 列表（参考）

| MCP | 功能 | 用途 |
|-----|------|------|
| mdold | 明道协作时代 v1 API | 任务、日历、消息、群组 |
| mingdao | 明道云 HAP API | 工作表记录增删改查、工作流触发 |
| claude.ai Gmail | Google Gmail | 邮件读取、发送、搜索 |
| claude.ai Google Calendar | Google 日历 | 日程查看、创建 |
| tencent-ad | 腾讯广告 | 广告投放管理 |
| ticktick | TickTick | 任务管理 |

### MCP 与 Skills 的关系

| 对比 | Skills | MCP |
|------|--------|-----|
| 定位 | 工具封装 | 协议标准 |
| 灵活性 | 固定流程 | 可连接任意数据源 |
| 使用场景 | 通用功能 | 接入外部 API |
| 关系 | 可调用 MCP | 被 Skills 调用 |
