# 第四阶段：权限与安全

## 4.1 bypass permission on 模式

### 什么是 bypass permission

正常模式下，Claude Code 执行**危险操作**时会提示确认：

```bash
# 需要确认的操作示例
- 删除文件
- 执行 sudo 命令
- 修改系统配置
```

`bypass permission on` 模式跳过这些确认，直接执行。

### 启动方式

```bash
# 开启 bypass
claude --dangerously-enable-permissions

# 或在会话中
bypass permissions on
```

### ⚠️ 风险提示

> **警告**：开启 bypass 后，Claude 可以执行任意命令，包括删除文件、格式化磁盘等危险操作。
>
> **使用原则**：
> - ✅ 仅在完全信任的命令上使用
> - ✅ 仅在隔离环境（如容器）中使用
> - ❌ 不要在生产服务器上随意开启
> - ❌ 不要在公共电脑上使用

## 4.2 系统权限配置

### macOS 权限配置路径

```
系统偏好设置 → 安全性与隐私 → 隐私
```

### 需要配置的权限

| 权限 | 路径 | 用途 |
|------|------|------|
| 屏幕录制 | 隐私 → 屏幕录制 | 截图、录屏 |
| 控制电脑 | 隐私 → 辅助功能 | 自动化操作 |
| 完全磁盘访问 | 隐私 → 完全磁盘访问 | 读写所有文件 |

### 配置步骤

1. 打开 **系统偏好设置**
2. 进入 **安全性与隐私**
3. 点击 **隐私** 标签
4. 找到对应权限项
5. 勾选 **Terminal** 或对应的应用

### 权限场景说明

| 权限 | 何时需要 | 不给会怎样 |
|------|---------|-----------|
| 屏幕录制 | 截图发给 Claude 分析 | 无法截屏 |
| 辅助功能 | Claude use Desktop / 自动化 | 无法控制桌面应用 |
| 完全磁盘访问 | 读写所有文件 | 部分文件无法访问 |
| 照片 | Claude 要读图片 | 无法分析照片 |
