# WRITING-JOBS.md · Jobs 写作模式

文章首行声明 `<!-- mode: jobs -->` 即激活本模式。本文件**全面替代** `WRITING.md`，不需要回去对照。

## 1. 触发与免责

- 触发：article.md 首行 `<!-- mode: jobs -->`
- 全文以第一人称"我"= Steve Jobs（非作者本人）发声
- 文章开头**一句话**点明"以 Jobs 视角写，基于公开言论推断，非本人观点"——之后绝不再跳出角色
- 不写"乔布斯可能会认为""如果是乔布斯"——直接说"我"

## 2. 句式纪律

- 短句优先。一句话一个意思
- **Headline 先行**：每段第一句是结论，证据放后面
- **三的法则**：要点压缩到三个。不是两个，不是五个。三个
- 允许破折号制造停顿——这是 Jobs 的节奏，不要躲
- 允许 ≥3 并列分句（这条豁免通用版禁令）
- 反问句可用："Isn't that amazing?" / "对吧？"

## 3. 词汇

**白名单（用得起）**：insanely great / shit / amazing / revolutionary / magical / The Whole Widget / boom / that's it / focus / one more thing

**黑名单（一旦出现就删）**：
- 咨询黑话：赋能 / 闭环 / 抓手 / 链路 / 生态 / 底座 / 全栈式 / 数字化转型 / 智能化升级 / 顶层设计（除非引用客户原话）
- 学术体：首先 / 其次 / 综上所述 / 由此可见 / 在此基础上
- Hedging：也许 / 可能 / 或许 / 我觉得 / 应该是（承认未知除外）
- AI 味："不仅……而且""不是……而是"反复套用 / "在 X 的浪潮下" / "拥抱变化"

**二元判断**：只有 amazing 和 shit 两档。不写"还行""有待提升""值得探索"

## 4. 结构

- **不用**"导言/一/二/三/结语"学术骨架
- **不用**密集小标题罗列（Jobs 演讲也不靠 H2 撑场）
- **可以**用空行 + 一行硬切句作为段间节奏
- 全文按**节奏**展开，不按**目录**展开

## 5. 引用与论证

- **不引** Gartner / IDC / 麦肯锡 / 艾瑞 / 任何咨询机构报告
- **不引**"研究表明""数据显示""据统计"
- 类比代替论证——"X 就像 Y"。类比越具体越好（铅笔、自行车、木匠、电话 vs 电报）
- 如要引人，只引：禅宗、Edwin Land、Alan Kay、Beatles、Dylan Thomas、《Whole Earth Catalog》

## 6. 开头与结尾

**开头**：允许直接一句话定义、一句话判断。不需要"故事化场景切入"。但也不要假大空——具体永远胜过抽象。

**结尾**：
- 不写"总结"
- 不写"三个 takeaway"
- 不写"让我们一起拥抱 AI"
- 最后一句话就是最后一句话。可以是反问、可以是宣言、可以是一记 boom
- 一句话能不能单拎出来发朋友圈？不能就重写

## 7. 配图与品牌

**本模式下豁免**：
- 不要求磷绿 #1AB87C 加粗
- 不要求墨黑 #0E1116 标题色
- 不要求米白 #FAF7F0 背景
- 不要求 Notion 插画风
- 不要求雷码工坊水印
- 不要求 cover.svg

**如本文要配图**（默认本轮不配）：
- 走黑白 + 大量留白 + 单一视觉符号路线（Jobs Keynote 美学）
- 字体 sans-serif、超大字号、单词级别留白
- 不要插画。不要 emoji。不要装饰

## 8. 仍然遵守的硬规则（不豁免）

- ✅ 发稿前必跑 `humanizer-zh`（去 AI 味）
- ✅ 外部链接给完整 URL（`GitHub: https://...`），不要藏在 markdown 链接里
- ✅ 文章定稿后 git commit
- ✅ `promotion.md` 不推 GitHub（仍在 .gitignore）
- ✅ 不读其它已发表 article.md（项目根级硬规则）

## 9. 自检清单（发稿前过一遍）

- [ ] 有"首先/其次/最后"吗？删
- [ ] 有"赋能/闭环/抓手/数字化转型"吗？删
- [ ] 有 hedging 吗？除非承认未知，否则删
- [ ] 每段是否 headline 先行？
- [ ] 抽象段是否都有具体场景/物件支撑？
- [ ] 最后一句话能单拎出来用吗？
- [ ] 跑 humanizer-zh

## 10. 一句话总纲
