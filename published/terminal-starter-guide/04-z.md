# 04 · z 目录跳转 / Directory Jumping with z

> 告别无尽的 `cd` 链，一个命令跳到任意常用目录。  
> No more typing long `cd` chains — jump to any frequently-used directory with a single command.

---

## 它解决什么问题 / What Problem It Solves

平时你可能需要这样进入一个项目：  
Without `z`, navigating to a project looks like this:

```bash
cd ~/Documents/coding/my-company/backend/api/src
```

用 `z` 之后，只需：  
With `z`, you just type:

```bash
z api
```

`z` 会记录你的 `cd` 历史，根据**访问频率 + 时间**加权，找到最可能的目标目录并直接跳过去。  
`z` tracks your `cd` history and uses **frecency** (frequency + recency) to guess the best match.

## 安装 zoxide / Install zoxide

```bash
brew install zoxide
```

然后在 `~/.zshrc` 末尾添加：  
Add to the end of `~/.zshrc`:

```bash
eval "$(zoxide init zsh)"
```

重新加载配置：  
Reload your shell config:

```bash
source ~/.zshrc
```

---

## 基本用法 / Basic Usage

> 注意：`z` 需要先"学习"，使用 `cd` 正常导航一段时间后，才能跳转。  
> Note: `z` needs to "learn" first. Use `cd` normally for a while, then it can start jumping.

### `z <关键词>` — 跳转

```bash
z project        # 跳到最常访问的包含 "project" 的目录
z doc api        # 多关键词：包含 "doc" 和 "api" 的目录
z pro            # 前缀匹配也可以
```

### `z -` — 返回上次目录

```bash
z -
```

### `zi` — 交互式选择（结合 fzf）

```bash
zi              # 列出所有记录的目录，用 fzf 交互式选择
zi project      # 列出匹配 "project" 的目录，交互式选择
```

> `zi` 需要安装 fzf，详见 [第 05 章](./05-fzf.md)。

### `z ..` — 跳到父目录（支持多级）

```bash
z ..            # 等同于 cd ..
z ../..         # 上两级
```

---

## 查看学习记录 / View Database

```bash
zoxide query -l         # 列出所有记录的目录（按权重排序）
zoxide query project    # 查看匹配 "project" 的目录
```

---

## 删除记录 / Remove Entry

如果某个目录不再存在或不想被推荐：  
To remove a stale or unwanted entry:

```bash
zoxide remove /path/to/old/directory
```

---

## 工作原理 / How It Works

每次你用 `cd` 进入一个目录，zoxide 都会记录：  
Every time you `cd` into a directory, zoxide records:

- 路径 / path
- 访问次数 / visit count
- 最近访问时间 / last access time

然后用公式计算 **frecency（频近度）**，执行 `z <关键词>` 时跳到得分最高的匹配目录。  
It then calculates **frecency** and jumps to the highest-scoring match.

## 完整 .zshrc 配置示例

```bash
# zoxide（放在 .zshrc 末尾）
eval "$(zoxide init zsh)"
```
