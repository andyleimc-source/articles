# 第五阶段：开发工具链

## 5.1 Git 与 GitHub

### Git 核心概念

| 概念 | 说明 |
|------|------|
| **仓库（Repository）** | 项目的版本数据库 |
| **提交（Commit）** | 一次完整的改动记录 |
| **分支（Branch）** | 独立的开发线 |
| **合并（Merge）** | 将分支合并回主线 |

### Git 工作流

```
工作目录 → 暂存区 → 本地仓库 → 远程仓库
  (修改)    (git add)  (git commit)  (git push)
```

### GitHub 账号注册

1. 访问 https://github.com
2. 输入用户名、邮箱、密码
3. 邮箱验证（国内邮箱可能收不到，推荐用 Gmail）
4. 选择计划（Free 足够）

### SSH 密钥配置

让本地 Git 与 GitHub 安全通信：

```bash
# 1. 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your.email@example.com"

# 2. 一路回车，使用默认位置

# 3. 查看公钥
cat ~/.ssh/id_ed25519.pub

# 4. 复制公钥，添加到 GitHub
#    GitHub → Settings → SSH and GPG keys → New SSH key
```

### 常用 Git 命令速查

```bash
# 初始化仓库
git init

# 克隆远程仓库
git clone https://github.com/user/repo.git

# 查看状态
git status

# 添加文件到暂存区
git add filename.txt      # 单个文件
git add .                 # 所有改动

# 提交
git commit -m "提交说明"

# 查看提交历史
git log --oneline

# 推送远程
git push origin main

# 拉取更新
git pull origin main

# 创建分支
git branch feature-name

# 切换分支
git checkout feature-name

# 创建并切换
git checkout -b feature-name

# 合并分支
git merge feature-name
```

### GitHub 协作流程（Pull Request）

```
1. Fork 仓库到自己的账号
2. Clone 到本地
3. 创建新分支
4. 在分支上修改代码
5. 提交并 push 到自己的 GitHub
6. 在 GitHub 上发起 Pull Request
7. 等待代码审核
8. 审核通过后合并到主仓库
```

## 5.2 前端三剑客（入门）

> 💡 说明：此处仅作概念扫盲，帮助理解网页开发的基础，不做深度编程教学。

### HTML — 网页结构

**HTML = 超文本标记语言**，定义网页的内容结构。

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>我的网页</title>
</head>
<body>
    <!-- 标题 -->
    <h1>一级标题</h1>
    <h2>二级标题</h2>

    <!-- 段落 -->
    <p>这是一个段落。</p>

    <!-- 链接 -->
    <a href="https://example.com">访问 Example</a>

    <!-- 图片 -->
    <img src="photo.jpg" alt="图片描述">

    <!-- 列表 -->
    <ul>
        <li>列表项一</li>
        <li>列表项二</li>
    </ul>
</body>
</html>
```

**常用 HTML 标签速查：**

| 标签 | 作用 |
|------|------|
| `<h1>` - `<h6>` | 标题（一级最大） |
| `<p>` | 段落 |
| `<a>` | 链接 |
| `<img>` | 图片 |
| `<ul>` / `<ol>` | 无序列表 / 有序列表 |
| `<div>` | 区块容器 |
| `<span>` | 行内容器 |
| `<input>` | 输入框 |
| `<button>` | 按钮 |
| `<table>` | 表格 |

### CSS — 网页样式

**CSS = 层叠样式表**，控制网页的外观。

```css
/* 选择器 { 属性: 值; } */

/* 元素选择器 */
h1 {
    color: blue;
    font-size: 24px;
}

/* class 选择器（以 . 开头）*/
.title {
    color: #333;
    text-align: center;
}

/* ID 选择器（以 # 开头）*/
#header {
    background-color: black;
}

/* Flexbox 布局 */
.container {
    display: flex;
    justify-content: space-between;
    align-items: center;
}
```

**CSS 三种引入方式：**

```html
<!-- 1. 内联样式（直接写在标签里） -->
<p style="color: red;">红色文字</p>

<!-- 2. 内部样式表（写在 head 里） -->
<head>
    <style>
        p { color: blue; }
    </style>
</head>

<!-- 3. 外部样式表（推荐） -->
<head>
    <link rel="stylesheet" href="style.css">
</head>
```

### JavaScript — 网页交互

**JavaScript** 给网页添加动态行为。

```javascript
// 变量声明
let name = "Andy";
const age = 30;  // const 表示常量，不能修改

// 函数
function greet(person) {
    return "你好，" + person + "!";
}

// 条件判断
if (age >= 18) {
    console.log("成年人");
} else {
    console.log("未成年");
}

// DOM 操作：获取元素并修改
document.querySelector("h1").textContent = "Hello World";

// 事件监听：点击按钮
document.querySelector("button").addEventListener("click", function() {
    alert("按钮被点击了！");
});
```

**JavaScript 在 HTML 中的三种引入方式：**

```html
<!-- 1. 内联（直接写在标签） -->
<button onclick="alert('点击')">点我</button>

<!-- 2. 内部脚本 -->
<script>
    console.log("页面加载完成");
</script>

<!-- 3. 外部文件（推荐） -->
<script src="app.js"></script>
```

## 5.3 语言安装库的概念

### 什么是包管理器

包管理器是**自动下载、安装、更新**代码库的工具：

```
无包管理器：手动下载 → 手动复制到项目 → 手动更新
有包管理器：一条命令搞定 → 自动处理依赖 → 一条命令更新
```

### Python — pip

```bash
# pip 是 Python 的官方包管理器

# 安装一个包
pip install requests

# 安装指定版本
pip install requests==2.28.0

# 从 requirements.txt 安装
pip install -r requirements.txt

# 查看已安装的包
pip list

# 升级包
pip install --upgrade requests
```

### Node.js — npm / yarn / pnpm

```bash
# npm 是 Node.js 官方包管理器（随 Node 一起安装）

# 初始化项目（生成 package.json）
npm init -y

# 安装包
npm install express       # 安装到 dependencies
npm install --save-dev eslint  # 安装到 devDependencies

# 全局安装（命令行工具常用）
npm install -g @anthropic-ai/claude-code

# 使用 yarn（需要单独安装）
npm install -g yarn
yarn add express

# 使用 pnpm（更快，磁盘更省）
npm install -g pnpm
pnpm add express
```

### 虚拟环境

不同项目可能需要不同版本的库，虚拟环境让每个项目有**独立的依赖空间**。

```bash
# Python 虚拟环境（venv）
python -m venv myenv          # 创建
source myenv/bin/activate     # 激活（Mac/Linux）
myenv\Scripts\activate        # 激活（Windows）

# 激活后 pip install 的包只在当前虚拟环境生效
pip install requests

# 退出虚拟环境
deactivate
```

### Go — go mod

```bash
# 初始化 Go 模块
go mod init github.com/user/project

# 添加依赖
go get github.com/gin-gonic/gin

# 更新依赖
go mod tidy
```

## 5.4 前后端概念扫盲

### 前端 — 用户界面

| 要素 | 说明 |
|------|------|
| **职责** | 展示界面、处理用户交互 |
| **技术** | HTML、CSS、JavaScript、React、Vue |
| **运行** | 浏览器 |
| **用户能看见** | ✅ |

### 后端 — 服务端逻辑

| 要素 | 说明 |
|------|------|
| **职责** | 处理数据、业务逻辑、API |
| **技术** | Python、Go、Java、Node.js |
| **运行** | 服务器 |
| **用户能看见** | ❌（除非出错） |

### 全栈协作模式

```
用户浏览器
    ↓ (HTTP 请求)
前端 (React/Vue)
    ↓ (JSON API)
后端 (Python Go Java)
    ↓
数据库 (MySQL PostgreSQL)
```

### 前后端分离的好处

| 好处 | 说明 |
|------|------|
| 独立开发 | 前端和后端可以并行 |
| 技术灵活 | 前端用 React，后端用任意语言 |
| 并行开发 | 前后端约定好接口格式后各自开发 |
| 重用性强 | 同一套后端可以对接 Web、iOS、Android |
