Why

为什么学 Claude Code ？

不是更努力，而是更聪明

传统开发

  • 遇到报错：复制 → 粘贴 Google → 逐条尝试
  • 写文档：对照模板，手动填充内容
  • 调试：加 console.log → 刷新 → 改 → 再来
  • 陌生接口：读英文文档，逐行理解

Claude Code 辅助

  • 遇到报错：直接截图/贴给 AI，分析并给出修复方案
  • 写文档：描述需求，AI 生成初稿，你审核修改
  • 调试：让 AI 分析日志，给你疑点排序
  • 陌生接口：让 AI 先给你解读，再动手
What

Claude Code 是什么

Anthropic 官方出品的本地 AI 编程 CLI 工具

在终端里运行，直接访问你的项目文件、系统环境和网络，用自然语言指挥 AI 帮你写代码、读代码、改代码、跑命令。

💻

本地优先

代码在本地，不上传云端

🔗

工具调用

读写文件、执行命令、访问网络

🤝

伙伴协作

自然语言驱动，适合 Code Review

核心概念 ①

MCP — Model Context Protocol

一句话理解 MCP

AI 的"USB 接口"——统一协议，让 AI 能调用各种外部工具（数据库、API、文件系统）而无需每个工具单独适配。

MCP 之前

  • 每个工具 → 单独写插件/集成
  • 接口不统一，维护成本高
  • 工具之间数据不互通

MCP 之后

  • 一个协议 → 连接所有工具
  • 工具按需加载，动态发现
  • 上下文自动共享，数据流转顺畅

Claude Code 内置 MCP 客户端，只需配置 Server 地址即可使用

核心概念 ②

Skills — 封装好的工具流程

什么是 Skill

一系列预设的工具调用步骤 + 提示词模板，打包成一个"技能"，一键调用。比如「生成 PPT」Skill，包含：确定大纲 → 读取风格规范 → 生成内容 → 输出 HTML。

Skill 的优势

  • 开箱即用
  • 减少重复描述
  • 输出质量稳定

Skill 的来源

  • 内置 Skill（官方提供）
  • 社区 Skill（clawhub.ai）
  • 自建 Skill（SKILL.md）

Skill 示例

  • ppt-generator：生成 PPT
  • video-frames：视频截帧
  • 1password：密钥管理
核心概念 ③

上下文管理 — 持久记忆文件

CLAUDE.md

项目指令文件

Claude Code 启动时自动读取工作目录及父目录下的 CLAUDE.md，也读取全局 ~/.claude/CLAUDE.md。用于定义项目规范、编码风格、常用命令等。

示例：指定代码风格、测试框架、禁止操作的目录等约束。

memory/ 自动记忆

跨会话持久记忆

由记忆插件（如 superpowers auto-memory）管理，保存在 ~/.claude/projects/ 下。Claude 自动写入和读取，记录用户偏好、项目决策、行为反馈等。

示例：记录用户的服务器 IP、常用工具偏好、项目架构决策。

⚠️ 注意：SOUL.md、AGENTS.md、USER.md、HEARTBEAT.md 是 OpenClaw 体系的文件，不是 Claude Code 原生功能。

核心概念 ④

会话管理 — Main vs Sub

Main Session

主会话，一次只做一个任务，适合深度工作

  • 串行执行，一个任务做完再做下一个
  • 上下文完全共享
  • 适合复杂、需要多轮迭代的工作

Sub Session

子会话，并行跑多个任务，适合批量工作

  • 通过 Agent(task="...") 启动（Claude Code 原生）
  • 独立上下文，互不干扰
  • 适合同时做多件事（如同时读多个文件）

when to use?

需要同时分析多个模块 → Sub Session 并行  |  一个功能从头做到尾 → Main Session

🛠 核心工具命令

Bash

Bash(command)

对话内执行终端命令，读写文件、跑脚本、查日志

Agent

Agent(task="...")

启动并行子 Agent，独立任务隔离执行（Claude Code 原生）

⚠️ 安全

权限开通 & 安全边界

系统权限（辅助功能）

允许 Claude Code 控制其他应用、模拟按键、读取屏幕

路径：系统设置 → 隐私与安全性 → 辅助功能 → 添加 Terminal

全盘访问（可选）

允许访问所有文件，默认只读写当前工作目录

网络访问

访问 URL、发送请求，默认关闭，需要时单独授权

bypass permission

bypass permission on

跳过所有命令执行确认

⚠️ 仅限确认安全的命令使用！

安全建议

  • 工作区限定在项目目录
  • 陌生仓库先用 Read 不急着 Write
  • 敏感操作加确认步骤

MCP ①

mdold — 明道云 MCP

一句话

明道云官方 MCP，让 AI 直接操作任务、日历、消息、群组、审批等核心模块。无需手动调 API，AI 替你完成增删改查。

可用功能（按模块）

任务 · TASK

创建 / 更新 / 完成 / 删除任务

高频

日历 · CALENDAR

创建日程 / 邀请成员 / 修改时间

高频

消息 · CHAT

发送消息 / 获取会话列表

中频

群组 · GROUP

创建群组 / 获取成员 / 审核申请

中频

典型使用场景

  • "帮我把所有待验收的任务标记为完成"
  • "创建一个明天上午10点的日程，邀请张三和李四"
  • "把超过3天没更新的任务负责人找出来"
  • "在群里发一条通知：本周五下午3点开会"

配置方式：在 Claude Code 里添加 mcpServer 地址，填入明道云的机构代码即可。

生态

Claude Code vs OpenClaw — 各有所长

Claude Code

官方 CLI · 本地开发

  • 深度代码理解（Code Interpreter）
  • bypass 权限跳过所有确认
  • MCP 工具调用能力强
  • 最适合编程开发

选它：写代码、调试、重构、Code Review

OpenClaw

第三方 Agent · 全场景

  • 微信/飞书/Telegram 接入
  • 定时任务 + 心跳监控
  • 网页搜索 + 内容摘要
  • 多渠道消息推送

选它：日常助理、消息推送、定时任务、社交

✅ 两者互补共存，OpenClaw 负责日常流程和消息通道，Claude Code 专注代码开发

大多数人用 AI 问问题
