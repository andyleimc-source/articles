## 正文

你有没有发现，跟 Claude Code 聊着聊着，它就变慢、变笨、还开始忘事？

不是模型变了，是**上下文越堆越厚**。

LLM 本身是无状态的，每句新消息都要把**整段历史**重新发一遍给模型——会话越长，每次调用的 token 越多，又慢又贵，质量还下降。

分享 3 个亲测好用的习惯 👇

1️⃣ **盯着右下角百分比切会话**
- <50%　放心聊
- 50–70%　任务收尾就 /clear
- 70–80%　主动切、写 handoff
- >80%　必须切，不要赌 auto-compact

2️⃣ **恢复历史会话永远选 summary**
Resume full session 会把全部历史重新加载进来，冷启动 + 没缓存，瞬间吃掉用量配额。除非必须引用原文细节，选 summary 就对了。

3️⃣ **记忆系统四件套**
- `CLAUDE.md` 项目约定
- `Progress.md` 进展记录
- `handoff.md` 切会话前的交接文档
- 任务告一段落立刻 `/clear`

让记忆落到文件里，而不是泡在上下文里。新会话读一下文档就能接上，比拖着 100k tokens 的长会话高效十倍。
