# 图片说明

## cover.png（放在文章正文顶部）
深夜书案上一本打开的皮册，被烛光照亮，页面上有手写墨迹。周围有竹简、铜烛台、铜器。
- **作用**：承载"皮册"核心意象，既古典又带一丝现代科技暗示（微微的冷色调光晕）
- **论点关联**：对应标题"给 AI 一本皮册"，是整篇文章的视觉锚点

## image/fable-scene.png（寓言章节结尾）
三个人物的古风水墨场景：国王坐案前、谋士站立神色茫然、老工匠捧上皮册。
- **作用**：寓言的核心画面，把"转折点"（国王想到了做皮册）视觉化
- **论点关联**：紧接寓言结尾"聪明可以借用，记忆必须自备"那句金句

## image/modern-notebook.png（第二节 AI 失忆章节）
俯视视角：一本写满笔记的笔记本摊开在桌上，旁边是休眠状态的笔记本电脑、咖啡、钢笔。
- **作用**：把古代皮册意象平滑过渡到现代场景——"笔记本电脑睡了，笔记本没睡"
- **论点关联**：呼应"你关掉终端的那一刻，Claude 就入睡了"

## image/claude-md-layers@2x.png（第三节 业内实践）
三层布局结构图：全局 ~/.claude/CLAUDE.md → 项目根 /project/CLAUDE.md → 子目录 CLAUDE.md（懒加载）。
- **作用**：把 walk-up 机制和懒加载可视化
- **论点关联**：帮助读者看懂"为什么 Claude Code 的多层设计很漂亮"

## image/six-files@2x.png（第四节 六文件方案）
六文件协作地图：Claude 居中，六个文件环绕，每个文件标注名字、职责、更新时机。
- **作用**：整套轻量方案的一张脑图
- **论点关联**：读者扫一眼就能记住体系全貌，代替大段文字
