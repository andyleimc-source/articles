## 02 · 技巧 01 / 10（页 2 / 12）— /goal
- 命令名：`/goal`
- 副标题：让 AI 自己干到达成
- 🧠 运作原理：给一个完成条件，Claude 每跑完一轮就检查一次，没达成就继续迭代，直到满足或你喊停。不用你一句句催。
- 📌 案例示范（代码块）：
  ```
  /goal 所有测试通过
  ```
  → AI 反复改代码 + 跑测试，绿了才停
- 金句：从催一句动一下 → 自己干到底

## 03 · 技巧 02 / 10（页 3 / 12）— /rewind
- 命令名：`/rewind`
- 副标题：改崩了的后悔药
- 🧠 运作原理：Claude Code 给每次文件改动和每轮对话都打了本地快照。`/rewind` 打开检查点列表，代码和对话能一起回到任意一步。
- 📌 案例示范：
  ```
  /rewind
  → 选「重构之前」检查点
  ```
  → 改崩的文件一键还原
- 金句：大胆试，反正能回去 ✓

## 04 · 技巧 03 / 10（页 4 / 12）— /context
- 命令名：`/context`
- 副标题：看清 token 都被谁吃了
- 🧠 运作原理：把当前上下文按来源拆成彩色网格——CLAUDE.md、对话、工具定义、记忆各占多少一目了然，还给优化建议。
- 📌 案例示范：
  ```
  /context
  → MCP 工具定义吃了 30%
  ```
  → 关掉用不上的，腾出空间
- 金句：省 token，先看钱花哪

## 05 · 技巧 04 / 10（页 5 / 12）— /memory
- 命令名：`/memory`
- 副标题：管 AI 的长期记忆
- 🧠 运作原理：一个入口编辑各层级 CLAUDE.md（全局/项目/本地），开关自动记忆，审计 AI 到底记了你项目的什么。
- 📌 案例示范：
  ```
  /memory
  → 编辑项目 CLAUDE.md
  → 查看 AI 自动记的条目
  ```
  → 记错的删，规矩当场补
- 金句：AI 记什么，你说了算

## 06 · 技巧 05 / 10（页 6 / 12）— /resume
- 命令名：`/resume`
- 副标题：找回任何历史会话
- 🧠 运作原理：每段会话都存在本地。`/resume` 弹历史列表按名字挑；`claude -c` 直接接上当前目录最近那次，上下文原样还在。
- 📌 案例示范：
  ```
  claude -c     # 接上最近一次
  /resume       # 翻历史挑一段
  ```
  → 昨天聊一半，今天接着干
- 金句：关了终端，思路不丢

## 07 · 技巧 06 / 10（页 7 / 12）— /permissions
- 命令名：`/permissions`
- 副标题：少弹一半确认框
- 🧠 运作原理：给工具设 允许/询问/拒绝 规则。常用安全命令加进 allow，Claude Code 直接跑不再每次问，危险操作仍拦你。
- 📌 案例示范：
  ```
  /permissions
  allow: Bash(npm test)
  allow: Bash(git status)
  ```
  → 读类操作不再打断你
- 金句：该放行放行，该兜底兜底

## 08 · 技巧 07 / 10（页 8 / 12）— /agents
- 命令名：`/agents`
- 副标题：一个人变一个小队
- 🧠 运作原理：定义带专属说明的子代理（subagent），各有独立上下文窗口。任务匹配上时 Claude 自动派活，互不污染主线。
- 📌 案例示范：
  ```
  /agents
  → 建「测试编写」代理
  → 建「代码审查」代理
  ```
  → 脏活分给分身，主线只推进
- 金句：复杂活，自动分派

## 09 · 技巧 08 / 10（页 9 / 12）— /loop
- 命令名：`/loop`
- 副标题：让它自己重复跑
- 🧠 运作原理：把一个 prompt 或命令按你给的间隔反复执行；不给间隔就让它自己掌握节奏。盯状态、跑维护任务全自动。
- 📌 案例示范：
  ```
  /loop 5m 检查部署状态
  ```
  → 每 5 分钟看一眼，有变化才叫你
- 金句：重复的活，交给循环

## 10 · 技巧 09 / 10（页 10 / 12）— /background
- 命令名：`/background`
- 副标题：会话挂后台继续干
- 🧠 运作原理：把当前会话脱离终端，转成后台代理继续跑，终端立刻解放去干别的；回头 `/resume` 接回来看结果。
- 📌 案例示范：
  ```
  /background
  → 大重构挂后台跑
  → 你照常写别的代码
  ```
  → 长任务不占着终端干等
- 金句：让它后台跑，你别干等

## 11 · 技巧 10 / 10（页 11 / 12）— /code-review ultra
- 命令名：`/code-review`
- 副标题：云端多代理审代码
- 🧠 运作原理：把当前分支的 diff 丢到云端沙箱，多个审查代理并行从 bug、风格、效率多个角度过一遍，不占你本地上下文。
- 📌 案例示范：
  ```
  /code-review ultra
  → 多代理并行审当前分支
  ```
  → 合并前先让一队 AI 挑刺
- 金句：合并前，过一队审查官
