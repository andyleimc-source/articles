### 📱 微信公众号

**标题（18字）：**
Claude Code进阶命令老手必看

**正文（约 820 字）：**
很多人用了好几个月 Claude Code，停在"叫它写代码 + /plan + /compact"这几招。

其实它还藏着一批中级命令，用上之后工作方式会再变一次。以下 10 个，是会用之后最值得接着掌握的。

**① /goal — 设个目标，它自己干到底**
给一个完成条件，Claude 每跑完一轮就检查一次，没达成就继续迭代，直到满足或你喊停。`/goal 所有测试通过`，它会反复改代码、跑测试，绿了才收手。

**② /rewind — 一键回到改坏之前**
Claude Code 给每次文件改动和每轮对话都打了本地快照。输入 `/rewind` 打开检查点列表，代码和对话能一起回退到任意一步，重构翻车不用手忙脚乱地救。

**③ /context — 看清 token 都被谁吃了**
把当前上下文按来源拆成彩色网格，CLAUDE.md、对话、工具定义、记忆各占多少一目了然，还给优化建议。常发现 MCP 工具定义悄悄吃掉三成空间。

**④ /memory — 管理 AI 的长期记忆**
一个入口编辑各层级 CLAUDE.md、开关自动记忆、审计 AI 到底记了你项目的什么。记错的当场删，规矩当场补。

**⑤ /resume（-c）— 找回任何历史会话**
每段会话都存在本地。`/resume` 弹出历史列表按名字挑，`claude -c` 直接接上当前目录最近那次，上下文原样还在，昨天聊到一半的活今天接着干。

**⑥ /permissions — 配白名单，少弹一半确认框**
给工具设允许/询问/拒绝规则。把常用的安全命令加进 allow，Claude Code 直接跑不再每次问，危险操作仍然拦你。

**⑦ /agents — 建专用子代理，复杂活自动分派**
定义带专属说明的子代理，各有独立上下文窗口。任务匹配上时自动派活，脏活交给分身，主线不被污染。

**⑧ /loop — 定个间隔，让它自己重复跑**
把一个 prompt 按你给的间隔反复执行，盯部署状态、跑维护任务全自动，有变化才叫你；不给间隔就让它自己掌握节奏。

**⑨ /background — 整段会话挂后台继续干**
把当前会话脱离终端转成后台代理继续跑，终端立刻解放去干别的，回头 `/resume` 接回来看结果。长任务不再占着终端干等。

**⑩ /code-review ultra — 一键云端多代理审代码**
把当前分支的 diff 丢到云端沙箱，多个审查代理并行从 bug、风格、效率多个角度过一遍，不占你本地上下文。合并前先让一队 AI 挑刺。

这 10 个命令单独用都能省事，组合起来，Claude Code 才算真的用顺了。建议收藏备用。
