# 小红书发布包生成（xhs.md）

为指定文章目录的图卡系列生成 `<article-dir>/xhs.md`（**仅本地，不推 GitHub**）。规则全部来自 CLAUDE.md，这里固化成流程。

## 输入
- `<article-dir>`：文章目录（含已出好的图卡）。
- 若未给目录，问一句，停下。

## 硬限制（违反 = 重写）
1. **标题 ≤ 20 字**（含标点/空格/英文，所有可见字符）。
2. **正文 caption ≤ 1000 字**，目标压到 **≤ 900 字**留缓冲。
3. **反引流禁词全程禁止**（caption / tag / 提及图卡文字都算）：
   公众号、微信、加 v、扫码、二维码、B站、抖音同名、Telegram、外站完整 URL、
   「雷码工坊笔记」作为公众号名。
   合规 CTA 用：收藏 / 点赞 / 评论区聊 / 评论区扣 1 / 关注我 / 下篇见。
4. 标签 ≤ 10 个。

## 步骤
1. 读文章目录下的图卡文件名（仅文件名，确定数量与顺序；**不读 article.md 全文**，标题/钩子可问用户或从图卡 caption 取）。
2. 生成 `xhs.md`，必含字段：
   - **标题 2-3 选**，每条标注精确字数（≤20）。
   - **完整正文 caption**：前言 + 每张图一条短钩子 + 收束，≤900 字，文末标注精确字数。
   - **话题标签**（≤10）。
   - **发布顺序**：按文件名列出，明确封面。
   - **检查清单**（checkbox：字数复核 / 上传顺序 / 封面 / 标题 / caption / 标签 / 定时 / 预览）。
   - **备注**：图来源 / 模型 / 风格锚点 / prompt 路径。
3. **自检**：写完用 `python3 -c "print(len(open('...').read()))"` 思路，逐条数标题与 caption 字数，超标当场删。禁词列表过一遍。
4. 提醒用户：`xhs.md` 已在 .gitignore，不会推送。

## 产出体验目标
发图时：打开小红书 app → 上传图卡 → 打开 xhs.md → 三段复制粘贴 → 发布，5 分钟搞定。
